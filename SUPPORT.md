# Support

## Reporting bugs

Open a [GitHub Issue](https://github.com/micschr0/gradle-test-shard-plugin/issues/new?template=bug_report.yml) using the Bug report template. Suspected **lost test coverage** (a module skipped on every node) is the highest-severity issue — report it even if you cannot reproduce reliably.

## Requesting features

Open a [GitHub Issue](https://github.com/micschr0/gradle-test-shard-plugin/issues/new?template=feature_request.yml).

## Usage questions

Open a [GitHub Discussion](https://github.com/micschr0/gradle-test-shard-plugin/discussions) if enabled, otherwise file an issue with the `question` label.

## Security reports

See [SECURITY.md](SECURITY.md). Do not open public issues for security vulnerabilities.
